| SPEC-INNATE-MANA-COST-01 | Per-Innate mana drain at level 1 + per-level scaling curve | Spec'd | P1 | Locked 2026-04-18. Full spec in [magic.md §Drain Scaling Per Level (SPEC-INNATE-MANA-COST-01)](systems/magic.md#drain-scaling-per-level-spec-innate-mana-cost-01). **Level-1 drain rates (asymmetric by design role, anchored on Mage's 200-mana base pool with no regen and no density modifier):** Haste 13.33 mana/s → ~15s uptime (burst tool); Sense 6.67 mana/s → ~30s uptime (exploration tool); Fortify 3.33 mana/s → ~60s uptime (held stance). Armor is excluded — it has no drain. **Shared scaling curve:** `drain(L) = max(base_drain * 0.96^L, base_drain * 0.25)` — 4% compounded reduction per level, floored at 25% of base at level ~35. **Explicit non-coupling:** drain is NOT modified by floor density (design decision: Innate cost is the brain's processing cost, not an environmental cost). Closes `magic.md` Open Question Q1. Impl notes: each toggle Innate needs a `base_drain` config field (constants: `INNATE_HASTE_BASE_DRAIN = 13.33f`, `INNATE_SENSE_BASE_DRAIN = 6.67f`, `INNATE_FORTIFY_BASE_DRAIN = 3.33f`, mana/sec). No density query in the drain path — keep Innate tick independent of `DungeonIntelligence` / floor state. Feeds SPEC-INNATE-STACKING-01 (next spec) which will decide concurrency rules. |
| SPEC-INNATE-STACKING-01 | Toggle-Innate concurrency rule — stacking vs one-active cap | Spec'd | P1 | Locked 2026-04-18. **Decision: Option A — Free stacking** (no hard concurrency cap). Full rule + stacking-combinations table + mid-game worked example in [magic.md §Stacking Rule (SPEC-INNATE-STACKING-01)](systems/magic.md#stacking-rule-spec-innate-stacking-01). Haste, Sense, and Fortify may run in any combination simultaneously; Armor does not participate (always-on, no drain). **Combined drain is additive** — at level 1, all three = **23.33 mana/sec** (13.33 + 6.67 + 3.33), which burns a Mage's 200-mana pool in **~8.6 seconds** with no regen. Mid-game worked example: level-15 Innate investment (~60% of base drain), 400-mana pool, 12 mana/s regen from Attunement → all-three net drain ~2.0 mana/s, effectively sustainable indefinitely; this is the build-identity payoff for Mages who invest in Attunement/INT. Warriors (60 pool) and Rangers (100 pool) naturally pick one or two per situation. Closes `magic.md` Open Question Q5. **Phase B now complete.** Impl notes: drain aggregation is a sum over active toggle-Innates each frame: `totalDrain = sum(drain(L_i) for i in activeToggleInnates)`. No concurrency guard in toggle-activation code — just pay the drain. Mana-exhaustion handling stays per-frame (if `mana <= 0` after subtraction, force all toggle-Innates off simultaneously; do not pick one to preserve). No new data fields — each Innate already has its own toggle state per SPEC-INNATE-MANA-COST-01. UI consideration: HUD mana-drain indicator should show summed drain when multiple Innates active (small numeric badge showing total mana/sec, not per-Innate breakdown — keeps HUD uncluttered). |

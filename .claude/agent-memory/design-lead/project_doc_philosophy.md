Documentation is the single most important asset in this project because AI is the entire dev team. Specs are the source of truth; code is generated output.

**Why:** The user (non-developer product owner) cannot read code. Documentation is their only way to understand, direct, and verify what the AI builds. If docs are wrong or incomplete, the project drifts.

**How to apply:** Always treat spec quality as the highest priority. When writing or updating specs, remember they serve dual purposes: (1) human-readable game design reference for the product owner, (2) machine-readable implementation instructions for AI agents. Both audiences must be served in every doc.
